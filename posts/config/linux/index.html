tmux

基本使用

无法使用鼠标

本人在服务器centos7环境下使用tmux, 遇到了鼠标点击无法切换窗格和不能使用鼠标滚轮查看之前的信息， 查询资料后找到以下资料.

  • 修改~/.tmux.conf文件配置
setw -g mouse-resize-pane on
setw -g mouse-select-pane on
setw -g mouse-select-window on
setw -g mode-mouse on

把上面内容加上去，再启动tmux session, ctrl+b进入命令模式，:source ~/.tmux.conf即可生效.

Tip
这个配置文件里面还可以加其他有关tmux的操作，以后留着.

Kali攻击机

arp-scan

arp-scan是对局域网中的主机进行arp扫描，也可以用于指定的网段. 通过发送ARP请求来发现同一子网中的所有设备，列出它们的IP地址和MAC地址. 常用参数如下：

# -f 	从指定文件中读取主机名或地址
arp-scan -f ip.txt

# -l  从网络接口配置生成地址（扫描本地的局域网）
arp-scan -l

# -i  各扫描之间的时间差，单位ms
arp-scan -l -i 1000

# -r  重复扫描的次数
arp-scan -l -r 5

nmap 端口扫描

Nmap的核心是一个网络发现和安全审计工具.它用于发现网络上运行的设备，并检测它们正在运行的服务和操作系统. 功能有主机发现，端口扫描(TCP SYN扫描，TCP CONNECT)，操作系统检测(OS)，服务识别. 最主要的功能还是发现指定的IP上有哪些端口仍然是active

Discovering Devices on a Network

# -PR arp发现，使用arp数据包查找活动主机
nmap -PR 192.168.0.1/24  

# -sn 不扫描端口，只用ping
nmap -sn 192.168.0.1/24

Port Scanning Techniques

# -sS SYN扫描 -sU UDP扫描
nmap -sS -sU -p 1-1000 192.168.0.1  

# 扫描所有端口
nmap -p- [ip]

Service and Version Detection

# 执行中等强度的服务版本检测
nmap -sV --version-intensity 5 192.168.0.105

DDOS攻击和slowlories

The SlowHTTPTest is a highly configurable tool that simulates some Application Layer Denial of Service attacks by prolonging HTTP connections in different ways.

攻击机向服务器缓慢发送http header，那么此时服务器不会拒绝连接，等待攻击机发送接下来的数据，由此占用服务器资源. slowloris将尝试创建尽可能多的HTTP GET请求，以占用Web服务器允许的所有剩余连接.这些基于线程的Web服务器只有在完成这些GET请求后才会关闭它们的线程. 只要攻击者在超时之前向服务器提供数据，HTTP请求将保持打开状态.通过发送部分，而不是畸形的数据包，Slowloris可以很容易地通过传统的指令检测系统.

low bandwidth in slowlories
To this day, Slowloris is used around the world by hacktivists to thwart governments, as it uses very low bandwidth requirements to achieve its goal of service denial.

以slowlories为例，使用方法如下

-p, --port
Port of webserver, usually 80

-s, --sockets
Number of sockets to use in the test

-v, --verbose
Increases logging (output on terminal)

-ua, --randuseragents
Randomizes user-agents with each request

-x, --useproxy
Use a SOCKS5 proxy for connecting

--https
Use HTTPS for the requests

--sleeptime
Time to sleep between each header sent
